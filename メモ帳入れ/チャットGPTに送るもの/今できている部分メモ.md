# 今できている部分メモ

Unityで作っている、マス目ターン制ハクスラRPGの最小プロトタイプについて、現時点でできている部分のまとめです。

# 現在の実装状況

## 1. グリッド上の移動

プレイヤーはグリッド上を1マスずつ移動できます。

操作:

- `WASD`
- 矢印キー

移動に成功するとプレイヤーターンを消費し、その後に敵が1回行動します。

## 2. ターン制の基本処理

「プレイヤーが1回行動すると、敵も1回行動する」という基本のターン制処理が入っています。

現在ターンを消費する行動:

- 1マス移動
- 通常攻撃
- 固有アビリティ使用

## 3. ターンを消費しない、その場回転

プレイヤーは移動せずに向きだけ変えられます。

操作:

- `Shift + WASD`
- `Shift + 矢印キー`

その場回転はターンを消費しません。
そのため、敵は行動しません。

最近修正した点:

- 敵に密着されている状態でも、敵の方向へ安定してその場回転できるようにした
- `Shift` を押しながら方向キーを押す場合だけでなく、方向キーを押しながら `Shift` を押した場合も、その場回転として拾うようにした
- 壁や敵で移動できなかった場合でも、ターンを進めずに向きだけ画面へ反映するようにした

## 4. 通常攻撃

通常攻撃は、プレイヤーが向いている前方1マスに攻撃します。

操作:

- `Space`

前方1マスに敵がいればダメージを与えます。
前方に敵がいなくても、現在は攻撃行動としてターンを消費します。

## 5. 固有アビリティ

装備している武器の固有アビリティを使えます。

操作:

- `E`

現在はランタイム生成の仮アビリティとして、以下の3種類があります。

- 前方攻撃
- 周囲攻撃
- HP回復

MPが足りない場合は発動しません。
HP回復アビリティを使った場合は、回復量がログに表示されます。

## 6. 武器生成

武器をランダム生成する処理が入っています。

武器には以下の要素があります。

- ランク
- レベル
- 経験値
- 固有アビリティ
- 汎用アビリティ枠数
- 成長率
- 能力値

武器ランクは、以下の式をもとに生成されます。

```csharp
baseRank = Mathf.CeilToInt(dungeonDifficulty * currentFloor * 0.25f);
spawnRank = Random.Range(baseRank - 1, baseRank + 2);
spawnRank = Mathf.Clamp(spawnRank, 1, 20);
```

## 7. 武器能力値

武器には以下の能力値があります。

- HP
- MP
- 攻撃力
- 防御力
- 会心率
- 会心ダメージ
- 属性ダメージバフ
- 回避率

武器の能力値はランク帯ごとの範囲からランダム生成されます。

ランク帯:

- 1〜5
- 6〜10
- 11〜15
- 16〜20

## 8. 武器レベルと経験値

敵を倒すと、装備中の武器に経験値が入ります。

現在は敵を1体倒すと経験値が10入ります。
必要経験値は仮で `現在レベル * 10` です。

武器レベルは最大10です。

敵撃破時には、ログに以下が表示されます。

- 敵を倒したこと
- 装備武器に経験値 +10
- 武器がドロップしたこと

## 9. 武器ドロップ

敵を倒したときに、敵がいたマスへ武器が1つドロップします。

ドロップ武器は既存の `WeaponGenerator` を使って生成しています。
ドロップした武器はグリッド上に黄色い小さな四角で表示されます。

## 10. 武器比較・拾う/交換する/拾わない選択

以前は、プレイヤーがドロップ武器のあるマスに移動すると自動装備されていました。
現在は自動装備をやめて、武器比較と選択が入っています。

プレイヤーがドロップ武器のあるマスに移動すると、武器比較パネルが開きます。

選択中の操作:

- `B`: 空きがあればバッグに入れる
- `1`: バッグ1番の武器と床の武器を交換する
- `2`: バッグ2番の武器と床の武器を交換する
- `3`: バッグ3番の武器と床の武器を交換する
- `Q`: 拾わずに閉じる
- `R`: 戦闘リセット

選択中は、移動・通常攻撃・固有アビリティ使用を受け付けません。
選択中に敵ターンも進みません。
バッグに入れる/交換する/拾わない選択そのものはターンを消費しません。

重要な仕様:

- ドロップ武器を直接装備する操作は廃止
- 装備枠とバッグを別々には持たない
- 装備中の武器もバッグ内に残る
- バッグ内のどの武器を装備中にするかを選ぶ
- バッグから武器が消えた場合、その武器は装備中でもなくなる
- バッグが満杯のときに `B` を押すと、ログに「バッグがいっぱい」と出て、武器は床に残る
- バッグが満杯でも、`1`〜`3` で床の武器とバッグ内武器を交換できる
- 装備中のバッグ枠を床武器と交換した場合、新しくその枠に入った武器が装備中になる

比較パネルは、羊皮紙風の2カラム表示になっています。

羊皮紙内の表示:

- 左側: 現在装備中の武器ステータス
- 右側: 落ちている武器ステータス
- 差分表示は右側の落ちている武器ステータスだけに表示
- プラス差分は緑系
- マイナス差分は赤系
- 変化なしは茶系の中立色

バッグ比較は羊皮紙の中には入れず、羊皮紙の外側にある下の補助欄に分けています。
これにより、メインの比較は「今装備中 vs 床武器」だけに集中して見られるようにしています。

バッグ比較欄では、Bag[1]〜Bag[3]それぞれと床武器の差分を簡易表示しています。
バッグが満杯のときに、どのバッグ枠と交換するかを判断しやすくするための表示です。

比較表示している項目:

- ランク
- レベル
- 経験値
- 攻撃力
- 防御力
- HP
- MP
- 会心率
- 会心ダメージ
- 属性ダメージバフ
- 回避率
- 成長率
- 汎用アビリティ枠数
- 固有アビリティ名
- 固有アビリティ消費MP
- 固有アビリティ倍率

床武器には簡易評価も表示しています。

評価の例:

- 攻撃寄り
- 防御寄り
- アビリティ変更あり
- バランス型

装備変更で最大HP/MPが変わった場合、現在HP/MPが上限を超えないようにしています。
拾っただけで全回復する挙動にはしていません。

## 11. 最小インベントリ

装備武器とは別の「独立した装備枠」は作らず、バッグ内の武器を装備する方式の最小インベントリが入っています。

現在の仕様:

- バッグには最大3本まで武器を持てる
- 初期武器もバッグの1番目に入り、その武器が装備中になる
- 装備中の武器はバッグ内に残る
- 通常時に `1`、`2`、`3` を押すと、バッグ内の該当武器を装備する
- 数字キーで装備しても、武器はバッグから消えない
- 装備武器の経験値は、バッグ内の同じ `WeaponInstance` に入る
- 次フロア移動時、バッグ内武器と装備中バッグ番号は持ち越される
- `R` で戦闘リセットすると、バッグも初期状態に戻る

OnGUIには、バッグ内武器の一覧が表示されます。

表示内容:

- インベントリ数 / 最大数
- バッグ番号
- 装備中表示
- 武器ランク
- レベル
- 攻撃力
- 固有アビリティ名

## 12. 敵

敵はフロアテンプレート内の敵記号から出現します。
現在は3種類の敵がいます。

敵種:

- `Slime`
  - 標準ステータスの基本敵
  - 標準AI
  - 表示色は緑系
- `Bat`
  - HPと防御力が低い
  - 攻撃力が高い
  - 攻撃的AI
  - 表示色は紫系
- `Golem`
  - HPと防御力が高い
  - 攻撃力が低い
  - 鈍重AI
  - 表示色は灰色系

テンプレートで使う敵記号:

- `E`: Slime
- `B`: Bat
- `G`: Golem

敵種ごとの情報は `EnemyDefinition` で管理しています。

`EnemyDefinition` が持っているもの:

- 敵名
- 基本HP
- 攻撃力
- 防御力
- AIタイプ
- 表示色

現在の敵AI:

- Slime
  - `EnemyAiKind.Standard`
  - 今まで通りの標準AI
  - プレイヤーに隣接していれば攻撃する
  - 離れていればプレイヤーに近づく
  - 壁や他キャラで主方向へ進めない場合、別方向を試す
  - どこにも動けない場合は動かない
- Bat
  - `EnemyAiKind.Aggressive`
  - プレイヤーへ近づける候補マスを見て、距離が縮む方向を優先する
  - Slimeより少し詰まりにくく、攻撃的に近づく
  - 2歩移動はまだ実装していない
- Golem
  - `EnemyAiKind.Slow`
  - 2ターンに1回だけ行動する
  - 行動しないターンは移動も攻撃もしない
  - 行動するターンは、標準AIと同じく近づくか隣接攻撃する

AIタイプは `EnemyAiKind` で持っています。
最小実装ですが、Slime、Bat、Golemで行動傾向の違いが出るようになっています。

敵もグリッド上を1マスずつ移動します。

## 13. 壁・障害物・複数フロアテンプレート

グリッド上に、移動できない壁マスが追加されています。

現在は本格的な自動ダンジョン生成ではなく、複数の固定フロアテンプレートからランダムに1つ選んで盤面を生成しています。

フロアテンプレートは `FloorTemplate` と `FloorTemplateDatabase` で管理しています。

テンプレートで使う記号:

- `#`: 壁
- `.`: 床
- `P`: プレイヤー初期位置
- `E`: Slime初期位置
- `B`: Bat初期位置
- `G`: Golem初期位置
- `S`: 階段

実装されていること:

- `GridMap` が壁マスを地形情報として管理する
- 盤面外は通行不可扱い
- Play開始時に、複数の固定フロアテンプレートからランダムに1つ選ばれる
- `R` でリセットしたときも、もう一度ランダムにテンプレートが選ばれる
- `P` の位置にプレイヤーが配置される
- `E` の位置にSlimeが配置される
- `B` の位置にBatが配置される
- `G` の位置にGolemが配置される
- `S` の位置に階段が配置される
- 敵記号が複数あるテンプレートでは、その数だけ敵が生成される
- プレイヤーは壁マスへ移動できない
- 壁に向かって移動してもターンを消費しない
- 敵も壁マスへ移動できない
- 敵は壁や他キャラでふさがれている場合、別方向を試す
- それでも移動できない場合、その敵は動かない
- 通常攻撃は壁越しに当たらない
- 前方系アビリティも壁に阻まれる
- 壁マスは黒っぽい色で表示される
- 階段マスは緑色で表示される
- `R` でリセットすると、ランダムに選ばれた壁ありフロアで初期盤面に戻る

現在用意しているテンプレート:

- `Small Crossroad`
- `Broken Hall`
- `Bent Passage`

`BattleController.CreateDefault()` では、`FloorTemplateDatabase.PickRandomDefault()` でテンプレートを選び、そこから壁、プレイヤー位置、敵種、敵位置を生成しています。

## 14. 階段・次フロア移動

フロア内に階段マスが配置されています。

実装されていること:

- テンプレート内の `S` から階段位置を生成する
- プレイヤーが階段マスへ移動すると、次のフロアへ進む
- 次フロアへ進むと、フロア番号が増える
- 次フロアでは、再びランダムな固定テンプレートから盤面が生成される
- 次フロアでは、壁、敵、階段、ドロップ武器、武器選択状態が作り直される
- 敵は階段マスへ移動しない
- 階段の上では武器がドロップしない
- OnGUIに現在のFloor番号が表示される

階段の仕様として、敵を全滅させなくても次フロアへ進めます。
「敵を倒して武器ドロップを狙う」か「危険なら敵を無視して階段へ向かう」かをプレイヤーが選べる形にしています。

## 15. 階層による敵・武器の強さ調整

Floorが深くなるほど、敵が少しずつ強くなるようになっています。
このFloor補正は、Slime、Bat、Golemすべてに反映されます。

現在の敵ステータス補正:

- Floorが1深くなるごとに、敵HP +5
- Floorが1深くなるごとに、敵攻撃力 +2
- Floorが1深くなるごとに、敵防御力 +1
- Floor 1は今までと同じくらいの強さ

敵ステータスの補正値は `BattleController` の定数で調整できるようにしています。

武器ドロップも現在のFloorを使ってランク計算されます。

武器ランク計算:

```csharp
baseRank = Mathf.CeilToInt(dungeonDifficulty * currentFloor * 0.25f);
spawnRank = Random.Range(baseRank - 1, baseRank + 2);
spawnRank = Mathf.Clamp(spawnRank, 1, 20);
```

Floor移動時にはログに以下が表示されます。

- `Floor X に進んだ`
- 敵のHP / 攻撃力 / 防御力補正
- 武器ドロップ基準ランク

武器ドロップ時にも、`Floor X基準 / 基準ランク` がログに出るようになっています。

## 16. 敵ステータス表示

画面右上に、各敵の名前、現在HP / 最大HP、攻撃力、防御力、AI傾向が表示されます。

表示例:

- `Slime1: HP 55/55 ATK 10 DEF 3 / 標準`
- `Bat2: HP 38/38 ATK 15 DEF 1 / 攻撃的`
- `Golem3: HP 85/85 ATK 7 DEF 8 / 鈍重`

敵を倒すと、HP表示は「撃破」になります。

- バッグ内武器を装備したこと
